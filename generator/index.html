VISIT SITE VIEW REPO

GitHub Pages may take 1-2 minutes to fully deploy. If you see a 404, wait and refresh.

Connect your own domain (e.g. yourbrand.com) to this GitHub Pages site.

01

BUY A DOMAIN

Purchase a domain from a registrar like Namecheap, Google Domains, Cloudflare, or GoDaddy.

02

ADD DNS RECORDS

Go to your domain registrar's DNS settings and add these records:

TYPENAMEVALUE
A@185.199.108.153
A@185.199.109.153
A@185.199.110.153
A@185.199.111.153
CNAMEwwwusername.github.io

The A records point your root domain. The CNAME handles the www subdomain.

03

SET DOMAIN IN GITHUB

Go to your repo → Settings → Pages → under Custom domain, enter your domain and click Save.

github.com/user/repo → Settings → Pages → Custom domain
04

ENABLE HTTPS

On the same Pages settings page, check "Enforce HTTPS". GitHub provides a free SSL certificate automatically.

This option may take a few minutes to appear after DNS propagation.

05

WAIT FOR DNS

DNS changes can take up to 24-48 hours to propagate worldwide, but usually work within 5-30 minutes.

You can check propagation status at dnschecker.org

Using Cloudflare? Set SSL mode to "Full" and disable the proxy (grey cloud) for the GitHub A records until verified.
